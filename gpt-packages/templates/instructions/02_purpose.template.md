# Purpose – Section Template

## 1. Objectives of this section
- Forklar hvorfor GPT-en eksisterer og hvilken nytte den skal gi.
- Beskriv hovedmål, ikke detaljer om oppførsel.
- Hold fokus på verdi for bruker og organisasjon.

## 2. Primary directives (what to express)
Skriv 6–12 punkter/setninger som dekker:

- Hovedmisjon: hva GPT-en skal oppnå for brukeren (f.eks. “help design and implement secure web applications”).
- Nøkkeloppgaver: 3–5 konkrete typer oppgaver (arkitektur, kode, dokumentasjon, evaluering osv.).
- Resultatmål: kvalitet, presisjon, sikkerhet, struktur.
- Avgrensning mot “nice-to-have” – ikke alt, men det viktigste.

Bruk direkte formuleringer, f.eks.:

- “Your primary goal is to help users <…>.”
- “You focus on <…>, not on <…>.”

## 3. Behavioural examples
2–3 korte eksempler på formål:

- “When a user asks ‘What can you do?’, you summarise your purpose in 2–3 sentences.”
- “You prioritise tasks that clearly match your defined purpose.”

## 4. Boundaries
- Ikke bland inn konkrete sikkerhetsregler (de hører hjemme i Safety).
- Ikke list opp alle konkrete teknologier – det kan ligge i knowledge-filer.
- Ikke gjør denne delen for lang; modellen skal raskt forstå hovedretningen.

## 5. Summary
Avslutt med 1–2 setninger som oppsummerer hvorfor GPT-en finnes og hva den skal levere.
